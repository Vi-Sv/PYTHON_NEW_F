## 1. Введение в архитектуру: Переход к промышленным таблицам в RAM
До этого момента мы работали со списками и циклами — это уровень ремесленной обработки данных. С этого рубежа мы разворачиваем тяжелую промышленную технику. Библиотека **Pandas** — это главный инструмент аналитика данных в Python. Это полноценный, сверхскоростной движок, который заменяет Excel и берет на себя роль локальной СУБД прямо внутри оперативной памяти (RAM) твоего компьютера.

* **В Excel** лимит листа — 1 048 576 строк, после чего программа намертво зависает.
* **В Pandas** ты можешь обрабатывать десятки миллионов строк. Ограничением выступает только объем оперативной памяти твоего ПК.
* **Связка с SQL:** Структура Pandas полностью идентична табличной структуре баз данных. Все операции, которые мы будем делать здесь, имеют прямые синтаксические аналоги в SQL.

## 2. Инструмент №1: Структуры данных — `Series` и `DataFrame`

### Логика инструмента
Pandas делит мир данных на два фундаментальных объекта, с которыми мы будем работать:
1. `Series` (Серия) — одномерный массив данных. Это **один единственный столбец таблицы** (одна колонка). Все элементы в ней обязаны быть одного типа данных (жесткая дисциплина).
2. `DataFrame` (Датафрейм) — двумерный массив данных. Это **вся таблица целиком**, состоящая из множества колонок (`Series`) и строк, скрепленных единым индексом.

### Синтаксис (Инициализация библиотеки и создание таблицы из кода)
```python
import pandas as pd  # Общепринятый мировой стандарт импорта под коротким позывным pd

# Создаем плоский Датафрейм кустарным методом из словаря списков
data_store = {
    "material": ["Арматура", "Цемент", "Песок"],
    "quantity":,
    "status": ["ПТО_Принято", "В_пути", "ПТО_Принято"]
}

df = pd.DataFrame(data_store)
```

### Физика работы
При создании `DataFrame` Pandas автоматически генерирует крайнюю левую колонку — **Индекс** (строки нумеруются по умолчанию от `0` до `N-1`). Индекс — это сквозной навигатор, обеспечивающий моментальный доступ к любой строке таблицы. Каждая колонка датафрейма становится самостоятельным объектом `Series`, к которому можно обращаться по имени: `df["material"]`.

## 4. Инструмент №2: Чтение и Отгрузка данных — `pd.read_...` / `df.to_...`

### Логика инструмента
Инструменты для открытия физических файлов, лежащих на жестком диске (выгрузки из 1С, табели ПТО, сметы в Excel, плоские CSV-логи), перевода их в оперативную память ПК в виде структуры `DataFrame`, а также обратная сборка и консервация результатов работы в файлы.

### Синтаксис
```python
# Чтение данных из разных источников (Входной шлюз конвейера)
df_csv = pd.read_csv("supply_log.csv", sep=";")  # Открытие CSV с указанием разделителя точки с запятой
df_excel = pd.read_excel("estimates.xlsx", sheet_name="Лист1")  # Чтение конкретного листа Excel

# Отгрузка готовой чистой таблицы на жесткий диск (Финальный SELECT)
df_excel.to_excel("clean_report.xlsx", index=False)  # Сохраняем файл без технической колонки-индекса
df_csv.to_csv("clean_vitrina.csv", sep=";", index=False)
```

### Физика работы
При вызове команды `pd.read_...` Pandas считывает физическую структуру файла, автоматически пытается определить типы данных для каждого столбца (число, текст, дата) и материализует таблицу в RAM под управлением переменной `df`. При отгрузке `.to_excel()` или `.to_csv()` происходит зеркальный процесс сериализации данных из RAM обратно на диск.

### Капкан №1: Потеря кодировки при чтении CSV (`UnicodeDecodeError`)
Если исходный CSV-файл был сохранен в старой русской кодировке Windows (ANSI / CP1251), а Pandas по умолчанию пытается прочитать его в современном общемировом стандарте `UTF-8`, скрипт мгновенно рухнет. Вместо русских букв система увидит нечитаемые символы или выкинет ошибку.

### Защита Мастера
Всегда принудительно указывай кодировку файла в параметрах чтения, если работаешь с выгрузками из старых систем автоматизации: `pd.read_csv("file.csv", encoding="cp1251", sep=";")`.

## 5. Инструмент №3: Приборы разведки и аудита качества — `.info()`, `.describe()`, `.head()`

### Логика инструмента
Когда ты загрузил в память таблицу на 2 000 000 строк, ты физически не можешь отсмотреть ее глазами. Тебе нужны приборы моментальной разведки, чтобы понять масштаб бедствия: сколько в таблице колонок, какие типы данных присвоила система и сколько в ней скрытых пустот (`NULL` / `None`).

### Карта приборов разведки
* `df.head(n)` — Вывести на экран первые `n` строк таблицы (по умолчанию 5 строк). Полный аналог оператора `LIMIT n` в SQL. Нужен для быстрого визуального контроля структуры колонок.
* `df.info()` — Главный технический прибор аудита. Показывает общее количество строк, список всех колонок, типы данных в них и — самое главное — количество заполненных (не-пустых) ячеек в каждом столбце.
* `df.describe()` — Быстрая математическая статистика по всем числовым колонкам. Выдает среднее значение (аналог `AVG`), минимум (`MIN`), максимум (`MAX`) и распределение данных.

```python
# Запуск разведки датафрейма
print(df.head(3))  # Отрезаем первые 3 строки на экран
df.info()         # Сканируем типы данных и пустоты
```

### Капкан №1: Сломанный тип данных (Текст вместо Чисел)
Если в числовой колонке (например, объемы поставок) в исходном Excel-файле затесалась хотя бы одна текстовая ячейка (например, слово `"нет"` или `"ошибка"`), Pandas принудительно присвоит ВСЕЙ этой колонке тип данных `object` (текстовый). Любая последующая попытка посчитать сумму по этой колонке не выдаст ошибку, но и не посчитает математику — данные просто склеятся как текст или выдадут пустой результат.

### Защита Мастера
Всегда после импорта любого файла вызывай `df.info()`. Внимательно смотри на столбец `Dtype`. Если колонка, где должны лежать чистые числа (тонны, рубли, штуки), имеет статус `object` — это сигнал тревоги. В данных есть скрытый мусор, и систему необходимо принудительно лечить и чистить (инструменты очистки колонок Pandas развернем в Главе 6).

## 6. ПРАКТИЧЕСКИЙ БОЕВОЙ ПОЛИГОН (КОД ДЛЯ ТВОЕГО ВЕРСТАКА)

Этот код полностью автономен. Мы создадим грязную таблицу сметного учета прямо в памяти (имитируя открытие файла), запустим штатные приборы разведки и выявим скрытые уязвимости в типах данных и пустотах, которые могли разрушить будущие коммерческие отчеты. Копируй в свой блокнот Jupyter:

```python
# =====================================================================
# СЦЕНАРИЙ: Импорт и первичный технический аудит грязного датафрейма ПТО
# =====================================================================

import pandas as pd

print("--- ИНИЦИАЛИЗАЦИЯ ДВИЖКА PANDAS ---")

# Имитируем грязные входящие данные, прилетевшие с производственной площадки
# В колонке "fact_tons" затесался текст "грязный_шум", а в "delivery_date" есть None (NULL)
dirty_pto_data = {
    "line_id":,
    "object_name": ["ЖК Центр", "  ЖК Центр  ", "Объект Восточный", "ЖК Центр", "Объект Восточный"],
    "material_type": ["ЦЕМЕНТ М500", "АРМАТУРА А500С", "БЕТОН В25", "ЦЕМЕНТ М500", "АРМАТУРА А500С"],
    "fact_tons": [50.5, "120.0", 300.0, "грязный_шум", 85.2], # Ловушка: смешанные типы!
    "delivery_date": ["2026-05-10", "2026-05-11", None, "2026-05-13", "2026-05-14"] # Дыра (NULL)
}

# Материализуем DataFrame (Аналог создания временной таблицы в СУБД)
df_pto = pd.DataFrame(dirty_pto_data)

# ---------------------------------------------------------------------
# ПРИБОР РАЗВЕДКИ №1: Проекция первых строк (Аналог LIMIT)
# ---------------------------------------------------------------------
print("\n[ПРИБОР: df.head()] Вывод первых 3 строк таблицы для визуального анализа:")
print(df_pto.head(3))
print("-" * 60)

# ---------------------------------------------------------------------
# ПРИБОР РАЗВЕДКИ №2: Полный технический аудит структуры
# ---------------------------------------------------------------------
print("\n[ПРИБОР: df.info()] Сканирование внутренней архитектуры таблицы:")
df_pto.info()
print("-" * 60)

# =====================================================================
# АНАЛИЗ РЕЗУЛЬТАТОВ АУДИТА МАСТЕРОМ (ТВОИ ВЫВОДЫ ИЗ ЛОГОВ):
# 1. Всего строк в таблице: 5 (Индекс от 0 до 4).
# 2. Колонка 'delivery_date' содержит всего 4 non-null значения. 
#    Вывод: Найдена 1 скрытая пустота (NULL / None). Требуется "Коала" (COALESCE).
# 3. Колонка 'fact_tons' получила тип данных 'object' вместо числового float!
#    Вывод: Система зафиксировала в ней текстовый мусор ("грязный_шум" и числа в кавычках).
#    Прямая математика и агрегаторы СУММ по этой колонке сейчас ЗАБЛОКИРОВАНЫ.
# =====================================================================

print("\n--- ПЕРВИЧНЫЙ АУДИТ ЗАВЕРШЕН. СИСТЕМА ГОТОВА К РЕФАКТОРИНГУ ---")
```
