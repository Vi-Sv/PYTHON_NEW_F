## 1. Введение в архитектуру: Переход на конвейерную сборку
До этого момента мы обрабатывали списки вручную по индексам — это все равно что ловить каждого рабочего на стройке лично ради одной подписи. Циклы в Python — это **автоматический промышленный конвейер**. Мы загружаем в начало трубы массив данных (список строк, табель, выгрузку из ПТО), задаем одно общее правило обработки, и система сама последовательно прогоняет через него каждый элемент сверху вниз до самой последней строчки.

* **В SQL** эту роль выполняет движок СУБД, обрабатывая миллионы строк невидимым внутренним циклом.
* **В Python** мы сами запускаем этот конвейер через синтаксический блок `for`.

---

## 2. Инструмент №1: Построчный переборщик — Цикл `for`

### Логика инструмента
Цикл `for` берет коллекцию данных (список, словарь или диапазон чисел), создает временную переменную-бегунка, берет первый элемент, выполняет над ним блок кода с отступом, возвращается наверх, берет второй элемент, и так до тех пор, пока элементы не закончатся.

### Синтаксис
```python
# Базовый перебор списка материалов
materials = ["Цемент", "Арматура", "Щебень"]

for material in materials:
    print("Проверка на складе: " + material)
```

### Физика работы
* `materials` — исходный массив строк (коллекция).
* `material` — имя временной переменной-бегунка. На каждом витке (итерации) цикла в нее автоматически залетает следующее значение из списка. Ты волен назвать ее как угодно (`item`, `row`, `x`), но имя должно отражать суть.
* **Жесткая дисциплина отступов:** Всё, что написано внутри цикла сдвинутым вправо (4 пробела или Tab), выполняется циклически. Как только код возвращается на исходную левую вертикаль — цикл завершен, конвейер остановился.

## 3. Инструмент №2: Генератор числовых диапазонов — `range()`

### Логика инструмента
Когда у тебя нет готового списка строк, но нужно выполнить действие ровно N раз (или сгенерировать сквозные индексы строк для табеля), используется встроенный счетчик `range()`.

### Синтаксис
```python
# Выполнить команду ровно 5 раз (счетчик выдаст числа от 0 до 4)
for i in range(5):
    print("Итерация конвейера №: " + str(i))
```

### Физика работы
Функция `range(start, stop, step)` генерирует последовательность целых чисел на лету, экономя оперативную память. Она не создает тяжелый список в RAM целиком.
* `range(5)` — выдаст `0, 1, 2, 3, 4`. Правая граница **никогда не включается** в результат (стандартный закон Python).
* `range(1, 6)` — выдаст числа от `1` до `5`.
* `range(0, 10, 2)` — выдаст четные числа с шагом 2: `0, 2, 4, 6, 8`.

## 4. Инструмент №3: Команды перенаправления потока — `continue` и `break`

### Логика инструмента
В процессе перебора миллионов строк скрипт может наткнуться на критическую аномалию. Нам нужны рычаги управления конвейером на ходу:
1. `continue` — Срочный сброс текущего витка. Скрипт мгновенно бросает выполнение оставшихся строк кода внизу цикла, возвращается наверх и берет *следующий* элемент из списка. (Аналог пропуска шума).
2. `break` — Аварийная остановка конвейера. Полный выход из цикла. Скрипт уничтожает цикл и идет дальше по коду ниже. (Аналог стоп-крана при критической ошибке).

### Синтаксис
```python
for num in range(1, 100):
    if num == 13:
        continue # Пропустить чертову дюжину, идти к числу 14
    if num == 50:
        break    # Дошли до половины, полностью заглушить конвейер
```

### Капкан №1: Бесконечные циклы и зависание RAM
В Python существует второй вид циклов — `while` (выполнять, пока условие истинно). Если в коде `while` допустить ошибку и не изменять условие внутри, скрипт уйдет в бесконечный цикл. Он намертво зажмет процессор и сожрет оперативную память, вызвав зависание верстака.

### Защита Мастера
В аналитике данных в 99% случаев используется строго цикл `for`. Он работает только в границах переданной ему коллекции (или диапазона `range`), поэтому физически не способен стать бесконечным. Забудь про `while` на этапе обработки витрин данных.

## 5. ПРАКТИЧЕСКИЙ БОЕВОЙ ПОЛИГОН (КОД ДЛЯ ТВОЕГО ВЕРСТАКА)

Скопируй этот код в свой файл `python_master.ipynb` и запусти ячейку. Мы собираем твой первый сквозной конвейер. Скрипт имитирует автоматический аудит суточного табеля рабочих СМР (Массив ООО «ФракДжет-Строй»). 

Конвейер выполняет комплексный маневр: построчно перебирает сотрудников, зачищает их грязные имена, перехватывает опасную пустоту `None`, считает общую сумму выплат, отсекает прогульщиков через `continue` и бьет по тормозам через `break`, если обнаруживает критическую ошибку в данных.

```python
# =====================================================================
# СЦЕНАРИЙ: Сквозной автоматический аудит грязного табеля ИТР и СМР
# =====================================================================

print("--- ЗАПУСК АВТОМАТИЧЕСКОГО КОНВЕЙЕРА ТАБЕЛЯ ---")

# Грязный лог-табель, прилетевший со строительного участка
# Формат строки: [Порядковый_номер, Имя_рабочего, Отработано_часов_факт]
raw_timesheet = [
    [1, "  Иванов И.И. ", 8],
    [2, " Петров В.С.  ", 0],     # Прогульщик (0 часов) — нужно отсечь
    [3, "Сидоров А.П.", None],    # Опасная пустота (NULL) — ловушка!
    [4, "КРИТ_ОШИБКА_БАЗЫ", -999], # Поломка логирования — стоп-кран!
    [5, "Николаев К.Д.", 8]       # Эта строка не должна обработаться из-за аварии выше
]

hourly_rate = 500  # Ставка: 500 рублей в час
total_payout = 0   # Бак-накопитель для финальной суммы выплат (Аналог SUM)
valid_rows_count = 0 # Счетчик успешно обработанных сотрудников (Аналог COUNT)

# Запускаем построчный перебор табеля
for row in raw_timesheet:
    row_id = row[0]
    raw_name = row[1]
    raw_hours = row[2]
    
    # ШАГ 1: Ювелирная зачистка текстового шума
    clean_name = raw_name.strip().upper()
    
    # ШАГ 2: Защита Мастера — Проверка на критическую аварию логов
    if "ОШИБКА" in clean_name or (raw_hours is not None and raw_hours < 0):
        print(f"АВАРИЙНЫЙ СТОП-КРАН: Строка №{row_id} повреждена ({clean_name}). Конвейер заблокирован!")
        break # Полная остановка системы, строки ниже не обрабатываются
        
    # ШАГ 3: Перехватчик пустоты (Аналог "Коалы" / COALESCE)
    # Если прилетел None — временно считаем, что человек отработал 0 часов
    hours = raw_hours if raw_hours is not None else 0
    
    # ШАГ 4: Отсечение нецелевых строк (Пропускаем тех, кто не работал)
    if hours == 0:
        print(f"Строка №{row_id}: Рабочий {clean_name} отработал 0 часов. Пропуск строки.")
        continue # Скрипт бросает этот виток и летит наверх к следующему рабочему
        
    # ШАГ 5: Бизнес-логика (Расчет денег и накопление итогов)
    salary = hours * hourly_rate
    total_payout = total_payout + salary # Накапливаем сумму (SUM)
    valid_rows_count = valid_rows_count + 1 # Наращиваем счетчик (COUNT)
    
    print(f"Строка №{row_id}: Рабочий {clean_name} проверен. Выплата: {salary} руб.")

print("-" * 50)
print(f"ИТОГ КОНВЕЙЕРА:")
print(f"Успешно обработано сотрудников (COUNT): {valid_rows_count}")
print(f"Общая сумма к выплате на участок (SUM): {total_payout} руб.")
print("--- СИНХРОНИЗАЦИЯ ПОТОКА ЗАВЕРШЕНА ---")
```
