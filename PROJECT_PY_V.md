Глава 1. ОСНОВНЫЕ МЕХАНИЗМЫ И КАРТА ПАМЯТИ

Всякое действие в Python – через переменную.
Аналогично SQL, где каждому новому столбцу мы давали имя [AS]
Любая операция обязана иметь свою переменную.

Чтобы система могла обратиться к результату выполнения абсолютно любой операции, любой операции необходимо присвоить имя:
x = 1+2
y = x + (4 * 5)
	Присваивая имя, система буквально вешает ярлык на значение, который в свою очередь, является адресом значения. Такое имя называется ПЕРЕМЕННАЯ.
Обращаясь к переменной, мы напрямую обращаемся к тому значению, которое скрывается под переменной.
 
Глава 1. ОСНОВНЫЕ МЕХАНИЗМЫ И КАРТА ПАМЯТИ

Инструмент №1: Присвоение ярлыка (Переменная) — [=]
В Python переменная никогда не хранит само значение. Она хранит только ссылку на объект в памяти.
Логика (Физика процесса под капотом):
Оператор =, это не математическое равенство, а приказ системе: «Создай объект в памяти и приклей к нему ярлык».
1.	Система сначала вычисляет всё, что находится справа от знака =.
2.	В оперативной памяти (RAM) рождается изолированный объект со своим значением и уникальным адресом. Система автоматически определяет его тип (int, str, bool и т.д.) — это называется динамической типизацией.
3.	Слева от знака = пишется имя (переменная). Система не заталкивает значение внутрь переменной, она просто связывает имя с созданным объектом. Переменная — это ярлык с надписью, приклеенный к объекту для быстрого доступа к нему.

Архитектурный чертеж (Синтаксис):
x = 1 + 2 - Справа родился объект 3 (тип int), слева приклеен ярлык 'x'
y = x + (4 * 5) - Справа вычислено 23, приклеен ярлык 'y'

Инструмент №2: Вывод на экран — [print()]
Логика (Физика процесса под капотом):
print() — это встроенная функция-транслятор. Она берет любой объект вложенный в сокбки: (переменную, сырое число, текст или результат вычисления), принудительно переводит его в текстовый вид и выводит в консоль.
1.	print() не меняет объекты в памяти и не возвращает результат для дальнейших расчетов. Его цель — показать текущее состояние данных. Коротко говоря: Функция print() создана только для отображения.
2.	По умолчанию, если передать в print() несколько элементов через запятую, он автоматически склеит их через пробел.
3.	В конце каждого вызова print() система автоматически ставит скрытый символ переноса строки (\n). Каждая следующая команда print() будет писать с новой строки., вложенный (end="") в print, приклеит следующую строчку в одну:

print("Загрузка...", end="") # Следующий print приклеится в эту же строку

Архитектурный чертеж (Синтаксис):
sales = 1500
print(sales)  #выведет на экран значение, адресом которого является sales
print("Финальная выручка составила:", sales, "рублей.") #шаблон-конструкция: сырой текст мы прячем в двойные кавычки; значение переменной выводим, указывая саму переменную – в данном сценарии, переменная фиксирована.

Инструмент №3: Разведка типа объекта — [type()]
Логика (Физика процесса под капотом):
Поскольку в Python действует динамическая типизация (система сама решает, какой тип присвоить объекту при рождении), аналитику необходим инструмент контроля. type() — это функция-анализ.
1.	Она считывает скрытые системные метаданные объекта, на который указывает переменная.
2.	Возвращает четкий маркер принадлежности к классу (например, <class 'int'>, <class 'str'>, <class 'DataFrame'>).
3.	Используется как предохранитель при импорте «грязных» данных из внешних источников (CSV, Excel, SQL-базы), когда нужно проверить, чем именно заполнена колонка — числами или текстом.
Ловушка невидимого текста (Крах математики):
Визуально в переменной может лежать число, но система считает его текстом.
Защита Мастера: Перед математическими операциями с новыми данными всегда проверяй их тип через type(). Если пришел текст вместо числа — применяй принудительную переплавку типов.

Архитектурный чертеж (Синтаксис):
revenue = 500000.75
client_name = "ООО Вектор"
print(type(revenue))      # Выведет: <class 'float'> (дробное число)
print(type(client_name))  # Выведет: <class 'str'> (строка/текст)
# Проверка типа в условии (Если тип — строка, то...)
if type(revenue) is float:
    print("Данные требуют округления")
 
Глава 2. КАРТА ВСЕЛЕННОЙ ТИПОВ ДАННЫХ

Инструмент №4: Переплавка типов – type(peremen)
int(переменная)
float(переменная)
str(переменная)
bool(переменная)
Логика (Физика процесса под капотом):
Это функции принудительного изменения формата данных. Система берет существующий объект одного типа и на его основе создает в памяти абсолютно новый объект другого типа.
x=1 #1 – в формате str (текста)
b=int(x) # созданная переменная принимает значение, переводит в новый формат, приклеивая новый ярлык. Старый объект остается в системе, однако его стоит игнорировать, так как теперь существует новая переменная, ссылающаяся на значение корректного типа.
Типы данных:
1.	int() — целое число: отсекает дробную часть числа (без округления!) или превращает строку из чистых цифр в целое число.
2.	float() — дробное число: превращает целое число или строку из цифр (можно с точкой) в дробное число.
3.	str() — текст: заворачивает любой объект в текстовые кавычки. Безопасен всегда.
4.	bool() — булево значение: превращает объект в логическое значение (True/False).

Защита Мастера: Перед переплавкой строк всегда очищай их от мусора. Заменяй запятые на точки через .replace(',', '.') и срезай пробелы с помощью .strip()

Ловушка переплавки (инст.4): Взрыв грязной строки (ValueError):
Функции int() и float() не умеют думать. Если в текстовой строке есть хоть один нецифровой символ (пробел, запятая вместо точки, буква), система не сможет совершить переплавку и полностью остановит выполнение скрипта.
# Капкан!
user_age = "25 "  # В конце строки затаился невидимый пробел
age = int(user_age) # Падение! ValueError: invalid literal for int() with base 10: '25 '
raw_revenue = "1500,50"        # Использована запятая вместо точки
revenue = float(raw_revenue)   # Падение! ValueError: could not convert string to float: '1500,50'

 
Глава 2. КАРТА ВСЕЛЕННОЙ ТИПОВ ДАННЫХ

Инструмент №5: Предохранитель пустоты — [None]
Логика (Физика процесса под капотом):
None — это специальный тип данных (NoneType), который означает полное отсутствие значения. Это прямой и абсолютный аналог NULL в SQL.
None — это не ноль 0 (ноль — это число), не пустая строка "" (строка — это текст) и не ложь False (это логический тип). None — это физическая пустота, резервация места в памяти под объект, которого еще нет или который был стерт.

Защита Мастера: Для проверки None всегда используй оператор тождественности is или is not

Архитектурный чертеж (Синтаксис):
# Инициализация пустой переменной (данные придут позже)
client_email = None
# Проверка на пустоту (Аналог WHERE client_email IS NULL)
if client_email is None:
    print("Внимание: email не заполнен!")
# Проверка на наличие данных (Аналог WHERE client_email IS NOT NULL)
if client_email is not None:
    print("Отправляем отчет")

РАБОТА С ТЕКСТОМ (STR)

Инструмент №6: Конструктор f шаблонов — [f-строки]
Логика (Физика процесса под капотом):
Прямой, но более мощный аналог функции CONCAT() из SQL. Форматирование строк через префикс f перед кавычками — это быстрый механизм сборки динамического текста в Python. 
1.	При сборке f-строки, система на лету сканирует текст в поисках фигурных скобок {}.
2.	Всё, что находится внутри скобок (переменные, математические операции, вызовы функций), Python вычисляет, автоматически переплавляет в текстовый тип (str) и физически вклеивает в итоговую строку.
3.	f-строки выполняются на уровне компилятора, что делает их самым производительным способом конкатенации в Python.
Архитектурный чертеж (Синтаксис):
manager = "Иван"
sales_volume = 450000
# Сборка сообщения с автоматической переплавкой типов
report = f"Менеджер {manager} закрыл месяц с объемом {sales_volume} руб."
print(report)  # Выведет: Менеджер Иван закрыл месяц с объемом 450000 руб.
 
Глава 3. ИНСТРУМЕНТЫ ЮВЕЛИРА: 
РАБОТА С ТЕКСТОМ (STR)

Инструмент №7: Триммер пробелов — [.strip() / .lstrip() / .rstrip()]
Логика (Физика процесса под капотом):
Метод .strip() — это полный аналог функции TRIM() в SQL. Он создает в памяти копию исходной строки, полностью вырезая из ее начала и конца (но не из середины) все «мусорные» невидимые символы.
1.	Под нож попадают не только стандартные пробелы, но и скрытые технические знаки: переносы строк (\n), знаки табуляции (\t) и возвраты каретки (\r).
2.	Модификации: .lstrip() — режет грязь только слева (в начале строки), .rstrip() — только справа (в конце строки).
3.	Метод работает только со строковым типом данных (str).

Архитектурный чертеж (Синтаксис):
# Очистка грязных данных на входе
raw_email = "   analytics@company.com \n\t"
clean_email = raw_email.strip()
print(clean_email)  # Выведет чистый текст: "analytics@company.com"
# Точечное удаление конкретных символов с краев (передаем их аргументом)
raw_id = "ID_40404_"
clean_id = raw_id.strip("_")  # Результат: "ID_40404"

Защита Мастера: Используй базовый .strip() только для зачистки пробелов и системных переносов строк. Для безопасного удаления конкретных префиксов и суффиксов в Python есть специализированные точечные инструменты: .removeprefix("www.") и .removesuffix(".com").

Глава 3. ИНСТРУМЕНТЫ ЮВЕЛИРА:

Инструмент №8: Унификатор регистра — [.lower() / .upper() / .title()]
Логика (Физика процесса под капотом):
Методы изменения регистра строк — это полные аналоги SQL-функций LOWER() и UPPER(). Так как для Python символы "А" (заглавная) и "а" (строчная) имеют абсолютно разные цифровые коды в системе Unicode, они считаются разными объектами.
1.	.lower() — переводя все буквы в нижний регистр (маленькие).
2.	.upper() — переводит все буквы в верхний регистр (ЗАГЛАВНЫЕ).
3.	.title() — переводит первую букву каждого слова в верхний регистр, а остальные — в нижний.
4.	Как и любая иная операция через точкe, эти методы создают новый объект в памяти, оставляя исходную строку нетронутой.

Архитектурный чертеж (Синтаксис):
# Приведение к единому стандарту перед фильтрацией (Решение проблемы "Москва" vs "москва")
user_input = "МоСкВа"
clean_city = user_input.lower()  # Результат: "москва"

# Прямое сравнение "на лету" без перезаписи переменной
if user_input.lower() == "москва":
    print("Регион: Центр")
# Стандартизация ФИО для отчета
raw_name = "иванов иван"
print(raw_name.title())  # Выведет: "Иванов Иван"


Фундаментальное правило:
Без явного переприсваивания, любой метод (операция с точкой), например: метод .lower(), не может переписать саму переменную внутри памяти. Если вызвать метод просто «в воздух», данные в переменной останутся не тронутыми:

client_status = "VIP"
client_status.lower() #данные остались не тронутыми
print(client_status)
СИСТЕМА ОТОБРАЗИТ: VIP

Защита Мастера: Всегда явно переприсваивай результат работы любого метода той же или новой переменной: 
client_status = "VIP"
вариант 1 (перезапись): client_status = client_status.lower()
вариант 2 (новая переменная): new_client_status = client_status.lower()

Инструмент №9: Хирургическая замена — [.replace()]
Логика (Физика процесса под капотом):
Метод .replace() — это полный аналог функции REPLACE() в SQL. Он сканирует строку слева направо, находит и физически вырезает (то, что мы объявили в скобках), вставляя на их место новое значение.
1.	По умолчанию метод работает по принципу «найти и заменить абсолютно все совпадения».
2.	Если искомая подстрока не найдена, Python не выдаст ошибку, а просто вернет точную копию исходной строки без изменений.
3.	Третьим необязательным аргументом можно передать жесткий лимит — сколько первых совпадений нужно заменить.

Архитектурный чертеж (Синтаксис):
# 1. Замена разделителя для переплавки в float (из "1500,50" в "1500.50")
raw_amount = "1,500,250.75"
# Удаляем запятые-разделители тысяч, меняя их на пустоту ""
clean_amount = raw_amount.replace(",", "")  # Результат: "1500250.75"

# 2. Ограничение количества замен (меняем только первый пробел)
text = "раз два три"
limited_text = text.replace(" ", "_", 1)  # Результат: "раз_два три"

Ловушка регистра при замене (инст.9):
Метод .replace() абсолютно слеп к смыслу текста и чувствителен к регистру букв. Если ты попытаешься заменить слово, написанное с большой буквы, передав методу аргумент в нижнем регистре, замена молча проигнорирует цель. Данные останутся необработанными.
# Капкан!
report = "Итоговый НДС составил 20%"
clean_report = report.replace("ндс", "налог") 
print(clean_report)  # Выведет: "Итоговый НДС составил 20%" — ничего не изменилось!
Защита Мастера: Если данные пришли в разном регистре, сначала примени унификатор регистра (Инструмент №8), а уже затем делай замену: clean_report = report.lower().replace("ндс", "налог").
