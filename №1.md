## 1. Введение в архитектуру: Python как среда манипуляции данными
В мире SQL данные покоятся на изолированных островах-таблицах, а СУБД выполняет приказ, возвращая проекцию. В Python мир устроен иначе: здесь нет жестких физических ячеек, пока мы сами их не создадим. Python для аналитика — это инструмент сквозной автоматизации данных. Он позволяет мгновенно загрузить массивы информации в память компьютера, обработать их через цепочку алгоритмов и трансформировать в нужный бизнес-формат. Результат работы — точный, очищенный от ошибок финальный отчет, готовый для принятия решений.
* **Таблица в SQL** — это монолитная плита.
* **Скрипт в Python** — это сборочный цех, где данные находятся в движении.

## 2. Инструмент №1: Вывод данных на экран — `print()`
Базовый инструмент ведения логов и визуализации состояния системы на определенном этапе выполнения конвейера.

### Логика инструмента
`print()` не меняет данные, он копирует текущее значение переменной или текст из RAM и швыряет его в консоль (на экран) для визуального контроля оператором.

### Синтаксис
```python
print("Инициализация системы ПТО выполнена.")
print(variable_name)
```

### Физика работы
Система берет указанный объект, переводит его в текстовый формат на лету и отправляет в стандартный поток вывода. Перечисление элементов через запятую внутри `print()` автоматически склеивает их через пробел.

### Капкан №1: Попытка вывести несуществующий объект
Если ты скомандуешь напечатать переменную, которую еще не объявил в коде выше, конвейер мгновенно остановится с критической ошибкой `NameError`.

### Защита Мастера
Никогда не вызывай `print()` вслепую для промежуточных вычислений внутри тяжелых циклов на миллионы строк — это парализует RAM и процессор. Вывод на экран — это ручной тормоз. Используй его точечно, только для фиксации ключевых вех скрипта.

## 3. Инструмент №2: Ячейка памяти — Переменная (`=`)

### Логика инструмента
Переменная — это именной ярлык, приклеенный к конкретному куску данных в оперативной памяти. Оператор `=` (присваивание) — это приказ: «Взять данные справа, положить их в RAM и наклеить на них ярлык с именем слева».

### Синтаксис
```python
object_name = "ЖК Центр"
fact_quantity = 150
```

### Физика работы
В отличие от жестких колонок в SQL, где тип данных прописан в схеме таблицы раз и навсегда, в Python действует **динамическая типизация**. Тип определяется автоматически по содержимому. Написал текст в кавычках — переменная стала текстовой. Написал целое число — числовой.

### КАРТА ВСЕЛЕННОЙ ТИПОВ ДАННЫХ (БАЗОВЫЕ ТИПЫ)
1. **`int` (Целые числа):** Числа без дробного хвоста (`1`, `25`, `-100`). Используются для штук, ID, счетчиков.
2. **`float` (Плавающие числа):** Числа с дробной частью (`15.5`, `0.003`, `-7.1`). В Python `float` — это аналог `REAL` / `FLOAT` в СУБД. Они имеют ту же уязвимость округления в двоичной системе.
3. **`str` (Строки / Текст):** Любые символы, зажатые в одинарные или двойные кавычки (`"Цемент"`, `'Марка М500'`).
4. **`bool` (Логический тип):** Имеет всего два состояния: `True` (Истина / `1`) или `False` (Ложь / `0`). Рождается в результате операций сравнения (сито фильтрации).

## 4. Инструмент №3: Базовая математика и работа со строками

### Логика инструмента
Математика в Python выполняется мгновенно над переменными в RAM. 

### Синтаксис
```python
# Числовая математика
total = fact + extra
delta = plan - fact
total_weight = volume * density
average_price = total_cost / total_orders

# Строковая математика (Сцепка / Конкатенация — аналог CONCAT)
full_name = material + " " + grade  # Результат: "Цемент М500"
```

### Физика работы
* Оператор `/` (деление) в Python 3 **всегда** возвращает тип `float`, даже если делишь `4 / 2` (результат будет `2.0`).
* Оператор `+` для строк работает как клей-момент (конкатенация), сшивая их в один монолитный текстовый объект.

### Капкан №1: Межклассовый конфликт (Попытка склеить текст и число)
Если ты попытаешься выполнить операцию `full_title = "Заказ №" + 105`, скрипт упадет со свистом с ошибкой `TypeError`. Python — язык с **жесткой типизацией**. Он не будет угадывать за тебя, что ты хочешь сделать: прибавить число к тексту или перевести текст в число.

### Защита Мастера
Перед склейкой всегда принудительно переплавляй числовые типы в текст (см. Инструмент №6 — `str()`).

### Капкан №2: Деление на ноль (`ZeroDivisionError`)
Если переменная в знаменателе примет значение `0` или `0.0`, выполнение всего кода будет заблокировано.

### Защита Мастера
Внедряй предохранители до момента деления с помощью логических условий (см. Главу 2) или специализированной обработки исключений.

## 5. Инструмент №4: Продвинутые операторы деления (`//` и `%`)

### Логика инструмента
1. `//` (Целочисленное деление) — рубит дробный хвост, оставляя только целую часть от деления (аналог `TRUNC` или переплавки в `INT`).
2. `%` (Остаток от деления) — вычисляет то, что осталось «за бортом» после деления нацело.

### Синтаксис
```python
hours = 25
days = hours // 24     # Результат: 1 (целые сутки)
rem_hours = hours % 24  # Результат: 1 (оставшийся час)
```

### Физика работы
Операции выполняются на уровне процессора. Остаток от деления `%` — это идеальный инструмент для проверки четности чисел (если `число % 2 == 0`, то оно четное) или для циклического распределения задач по пулу рабочих.

## 6. Инструмент №5: Переплавка типов — Инструменты `int()`, `float()`, `str()`

### Логика инструмента
Принудительное изменение типа данных переменной на ходу (полный аналог `CAST(col AS type)` или скоростного Postgres-маневра `col::type`).

### Синтаксис
```python
# Из текста в целое число
fact_count = int("150")

# Из целого числа в плавающее
weight_float = float(50)  # Результат: 50.0

# Из числа в текст для безопасной сцепки (CONCAT)
log_message = "Обработано строк: " + str(rows_processed)
```

### Физика работы
Python берет внутреннее значение объекта, проверяет его на совместимость с целевым форматом, создает в RAM новый объект требуемого типа и перенаправляет на него ярлык переменной.

### Капкан №1: Грязные данные при плавлении текста в число
Если в строке окажется хотя бы один нечисловой символ (например, `"150 шт"` или `"150 "` с невидимым пробелом на конце), инструмент `int()` выдаст критическую ошибку `ValueError`.

### Защита Мастера
Перед очисткой и плавлением текстовых выгрузок из ПТО всегда задействуй методы зачистки шума (см. Инструмент №7 — `.strip()`).

## 7. Инструмент №6: Зачистка текстового шума — Методы `.strip()`, `.lower()`, `.upper()`

### Логика инструмента
Полные аналоги «Команды ювелиров» из SQL (`TRIM`, `LOWER`, `UPPER`). Уничтожают паразитные пробелы по краям строк и приводят символы к единому регистру для точного сопоставления.

### Синтаксис
```python
raw_material = "  Арматура А500С  "

# Комплексная зачистка «в один проход» (Маневр Ювелира)
clean_material = raw_material.strip().lower()
# Результат: "арматура а500с"
```

### Физика работы
Метод `.strip()` сканирует текстовую строку с левого и правого конца, откусывая пробелы, знаки табуляции (`\t`) и символы переноса строки (`\n`). Метод `.lower()` переводит все буквы в нижний регистр. Оригинальная переменная `raw_material` не меняется — метод возвращает чистую виртуальную проекцию, которую мы фиксируем в новую переменную.

### Капкан №1: Попытка применить текстовый метод к числу
Если в переменной лежит тип `int` или `float`, вызов метода `.strip()` или `.lower()` приведет к падению скрипта с ошибкой `AttributeError`. У чисел нет таких методов.

### Защита Мастера
Если колонка грязная и тип данных не определен, сначала принудительно оберни объект в `str()`, а затем вызывай методы зачистки: `str(variable).strip().lower()`.

## 8. Инструмент №7: Предохранитель пустоты — Работа с `None`

### Логика инструмента
`None` в Python — это абсолютное ничто, полная пустота (точный аналог `NULL` в SQL). Любая попытка выполнить математическую операцию или применить метод к `None` гарантированно убивает скрипт.

### Синтаксис
```python
fact_delivery = None
```

### Физика работы
`None` — это специальный тип данных (`NoneType`), обозначающий отсутствие значения или неопределенность.

### Капкан №1: Каскадное заражение пустотой
В отличие от SQL, где `100 + NULL = NULL` происходило тихо и без падения запроса, в Python операция `100 + None` порождает мгновенный `TypeError`, который полностью останавливает весь твой конвейер обработки.

### Защита Мастера
Перед проведением любых расчетов над импортированными данными всегда проверяй их на пустоту (инструменты проверки и аналог «Коалы» — `COALESCE` — будут детально развернуты в блоке управляющих конструкций и Pandas).

## 9. ПРАКТИЧЕСКИЙ БОЕВОЙ ПОЛИГОН (КОД ДЛЯ ТВОЕГО ВЕРСТАКА)

Скопируй этот код целиком в свой файл `python_master.ipynb` внутри VS Code и запусти ячейку (`Shift + Enter`). Изучи, как ведут себя грязные данные и как монолитно отрабатывают инструменты защиты.

```python
# =====================================================================
# СЦЕНАРИЙ: Очистка и калькуляция грязных данных по поставке цемента из ПТО
# =====================================================================

print("--- СТАРТ КОНВЕЙЕРА ОЧИСТКИ ДАННЫХ ---")

# Исходные грязные логи, прилетевшие из ручной Excel-таблицы
raw_title = "   ЦЕМЕНТ Марка М500   "
raw_box_weight = "50"        # Число застряло в текстовом формате
raw_delivered_boxes = "200"  # Еще один кусок грязного текста с пробелом

# ШАГ 1: Ювелирная зачистка текста и унификация регистра (Аналог TRIM + LOWER)
clean_title = raw_title.strip().upper()

# ШАГ 2: Переплавка типов (Аналог CAST / ::INT)
box_weight = int(raw_box_weight)
delivered_boxes = int(raw_delivered_boxes.strip())


# ШАГ 3: Базовая математика в RAM
# Проводим расчеты. Помни: деление через "/" всегда возвращает тип float.
total_weight_kg = box_weight * delivered_boxes
total_weight_tons = total_weight_kg / 1000  

# ШАГ 4: Сборка финального лога через принудительную переплавку в str()
# Защита от межклассового конфликта: плавим числа в текст перед сцепкой.
report_message = (
    "МАТЕРИАЛ: "
    + clean_title
    + " | ВЕС НЕТТО: "
    + str(total_weight_tons)
    + " ТОНН."
)

# ШАГ 5: Вывод отчета на экран
print(report_message)
print("--- СИНХРОНИЗАЦИЯ СИСТЕМЫ ЗАВЕРШЕНА ---")
