# Глава 6. ХИРУРГИЧЕСКИЙ ОТСЕВ И ОЧИСТКА: ФИЛЬТРАЦИЯ СТРОК (WHERE) И СЕЙВ ОТ ПУСТОТ (COALESCE) В PANDAS

## 1. Введение в архитектуру: Операции над осями в RAM
В СУБД мы отсекаем строки на дальних подступах к оперативной памяти через блок `WHERE`, чтобы процессор работал только со «скульптурой». В Pandas данные уже лежат в RAM. Наша задача — хирургически срезать лишний «камень», применяя быстрые векторные маски, зачистить скрытые пустоты и принудительно переплавить сломанные типы колонок в чистые числа.

---

## 2. Инструмент №1: Фильтрация строк через Логические маски — `df[маска]`

### Логика инструмента
Абсолютный аналог блока `WHERE` в SQL. Мы задаем условие к конкретной колонке. Pandas сканирует её, проверяет каждую ячейку и формирует невидимый логический щит (маску из `True` и `False`). Затем этот щит прижимается к датафрейму, и через него проскакивают только те строки, где условие выдало `True`.

### Синтаксис
```python
# Настоящий аналог: WHERE object_name == 'ЖК Центр'
mask = df["object_name"] == "ЖК Центр"
df_filtered = df[mask]

# Скоростной маневр в одну строку на автопилоте рук:
df_filtered = df[df["object_name"] == "ЖК Центр"]
```

### Физика работы
Операция выполняется векторизованно на уровне языка C, без медленных циклов Python. Исходный датафрейм `df` не разрушается — система создает в RAM его очищенную виртуальную проекцию `df_filtered`.

### Сочетание условий (Связки AND / OR)
В Pandas вместо слов `and` и `or` внутри фильтров используются жесткие побитовые операторы. **Каждое отдельное условие обязательно зажимается в круглые скобки!**
* `&` — Побитовое И (аналог `AND`). Сужает сито вычислений.
* `|` — Побитовое ИЛИ (аналог `OR`). Расширяет сито вычислений.

```python
# Аналог: WHERE (diameter == 89) AND (status == 'Принято')
df_clean = df[(df["diameter"] == 89) & (df["status"] == "Принято")]
```

## 3. Инструмент №2: Охотник за пустотой и Заплатка «Коалы» — `.isna()` и `.fillna()`

### Логика инструмента
В Pandas системная пустота (`NULL`) обозначается как `NaN` (Not a Number) для чисел или `None` для текста. 
1. Метод `.isna()` — детектор дыр (полный аналог `IS NULL` в SQL).
2. Метод `.fillna(value)` — перехватчик пустот и установка заплатки (абсолютный аналог оператора `COALESCE`).

### Синтаксис
```python
# 1. Найти все строки с дырами (Аналог: WHERE volume IS NULL)
df_holes = df[df["volume"].isna()]

# 2. Маневр "Коалы": Заплатка на дыру (Аналог: COALESCE(volume, 0.0))
df["volume"] = df["volume"].fillna(0.0)
```

### Физика работы
Метод `.fillna(0.0)` сканирует указанную колонку. Как только система натыкается на метку пустоты `NaN`, она выжигает её и вставляет на это место твое безопасное значение `0.0`. Это спасает будущую математику от каскадного обнуления.

## 4. Инструмент №3: Принудительная переплавка типов — `.astype()` и `pd.to_numeric()`

### Логика инструмента
Исправление сломанных типов колонок (полный аналог `CAST(col AS type)` или скоростного маневра `col::type`).

### Синтаксис
```python
# Классическая переплавка, если в колонке чистый текст-число
df["line_id"] = df["line_id"].astype("int64")

# Тяжелая артиллерия для грязных колонок с мусором:
df["fact_tons"] = pd.to_numeric(df["fact_tons"], errors="coerce")
```

### Физика работы и Капкан №1: Падение при плавке грязного текста
Если ты применишь метод `.astype("float64")` к колонке, где среди чисел сидит текстовый мусор (слово `"ошибка"` или `"нет данных"`), скрипт упадет. Конвейер заблокирован.

### Защита Мастера — Маневр с параметром `errors='coerce'`
Используй специализированный инструмент `pd.to_numeric(column, errors='coerce')`. 
* Физика маневра: Pandas начинает плавить элементы колонки в числа. Если он встречает чистую строку-число `"50.5"`, он превращает её во `50.5`. Если он натыкается на текстовый мусор `"грязный_шум"`, параметр `errors='coerce'` **принудительно превращает этот мусор в безопасную пустоту `NaN`**. Текст уничтожен. После этого мы мгновенно закрываем зазор методом `.fillna(0.0)`. Колонка спасена и переведена в числовой формат!

## 5. ПРАКТИЧЕСКИЙ БОЕВОЙ ПОЛИГОН (КОД ДЛЯ ТВОЕГО ВЕРСТАКА)

Этот код решает критические баги, которые мы обнаружили при разведке в Главе 5. Скопируй его в свой `python_master.ipynb` и запусти ячейку. 

Скрипт проводит хирургический ремонт таблицы ПТО: полностью ликвидирует текстовый мусор в тоннаже, преобразует типы данных, перехватывает пустые даты через маневр «Коалы» и отсекает лишний «камень» фильтрацией по условиям.

```python
# =====================================================================
# СЦЕНАРИЙ: Хирургическое лечение, переплавка типов и фильтрация датафрейма ПТО
# =====================================================================

import pandas as pd

# Загружаем ту самую грязную структуру из Главе 5
dirty_pto_data = {
    "line_id":,
    "object_name": ["ЖК Центр", "  ЖК Центр  ", "Объект Восточный", "ЖК Центр", "Объект Восточный"],
    "material_type": ["ЦЕМЕНТ М500", "АРМАТУРА А500С", "БЕТОН В25", "ЦЕМЕНТ М500", "АРМАТУРА А500С"],
    "fact_tons": [50.5, "120.0", 300.0, "грязный_шум", 85.2], 
    "delivery_date": ["2026-05-10", "2026-05-11", None, "2026-05-13", "2026-05-14"] 
}
df_pto = pd.DataFrame(dirty_pto_data)

print("--- СТАРТ РЕФАКТОРИНГА И ОЧИСТКИ ТАБЛИЦЫ ---")

# ШАГ 1: Лечение сломанной колонки 'fact_tons' (Маневр Мастера с errors='coerce')
# Мусор "грязный_шум" принудительно плавится в NaN
df_pto["fact_tons"] = pd.to_numeric(df_pto["fact_tons"], errors="coerce")

# Мгновенно закрываем образовавшуюся дыру заплаткой 0.0 (Аналог COALESCE)
df_pto["fact_tons"] = df_pto["fact_tons"].fillna(0.0)


# ШАГ 2: Заплатка на пустую дату доставки (Маневр "Коалы")
# Если дата утеряна — пишем системный маркер 'НЕТ_ДАТЫ'
df_pto["delivery_date"] = df_pto["delivery_date"].fillna("НЕТ_ДАТЫ")


# ШАГ 3: Ювелирная зачистка текстового шума в названиях объектов
# Чтобы строки склеивались идеально, убираем пробелы по краям
df_pto["object_name"] = df_pto["object_name"].str.strip()


# КОНТРОЛЬНЫЙ АУДИТ ПОСЛЕ ЛЕЧЕНИЯ
print("\n[АУДИТ] Структура таблицы после рефакторинга:")
df_pto.info()
print("-" * 60)


# ШАГ 4: Хирургический отсев строк (Аналог блока WHERE с двумя условиями)
# Задача: Оставить в отчете только поставки на 'ЖК Центр', где объем строго больше 0
mask_where = (df_pto["object_name"] == "ЖК Центр") & (df_pto["fact_tons"] > 0)
df_final_report = df_pto[mask_where]


print("\n[ФИНАЛЬНЫЙ СЛУЖЕБНЫЙ ОТЧЕТ] Скульптура данных готова к отгрузке:")
print(df_final_report)
print("--- СИНХРОНИЗАЦИЯ МОДУЛЯ ОЧИСТКИ ЗАВЕРШЕНА ---")
```
