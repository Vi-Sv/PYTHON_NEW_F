## 1. Введение в архитектуру: Зачем нужны коллекции?
До этого момента мы работали с одиночными переменными — это все равно что обрабатывать одну изолированную строку. Но на реальном производстве данные прилетают пакетами: списки сотрудников, массивы диаметров труб, справочники соответствия кодов ПТО. 
* **`List` (Список)** — это твоя вертикальная колонка в Excel или массив строк, где важен физический порядок.
* **`Dict` (Словарь)** — это инструмент создания неявных связей (маппинга), где по уникальному ключу (ID) система мгновенно вытягивает нужную характеристику.

## 2. Инструмент №1: Динамический конвейер — Список (`List`)

### Логика инструмента
Список — это упорядоченная последовательность элементов, зажатая в квадратные скобки `[]`. Он может хранить внутри себя объекты любых типов и динамически расширяться по ходу выполнения скрипта.

### Синтаксис
```python
diameters = [57, 89, 108, 219]
raw_data = ["Цемент", 50, 120.5, True]  # Разные типы в одном баке
```

### Физика работы: Индексация и Срезы (Аналог `LEFT`/`RIGHT`/`LIMIT`)
Каждый элемент в списке имеет свой порядковый номер — **индекс**. 
* **Внимание: В Python отсчет всегда начинается с `0`!** Первый элемент — `[0]`, второй — `[1]`.
* **Отрицательные индексы (Взгляд с тыла):** Индекс `[-1]` гарантированно вернет *последний* элемент в списке, `[-2]` — предпоследний. Это спасает, когда ты не знаешь точную длину массива.
* **Срезы (`Slices`):** Маневр `[start:stop]`. Извлечение куска массива. Правая граница `stop` **не включается** в результат (закон Python).

```python
pipes = ["Труба_57", "Труба_89", "Труба_108", "Труба_219"]
print(pipes[0])      # Результат: "Труба_57"
print(pipes[-1])     # Результат: "Труба_219" (последняя)
print(pipes[1:3])    # Результат: ["Труба_89", "Труба_108"] (индексы 1 и 2, 3-й не вошел)
```

### Модификация списка (Команды управления)
* `.append(element)` — Добавить элемент в самый конец списка (увеличить длину отчета вниз).
* `len(my_list)` — Посчитать общее количество живых строк в списке (полный аналог `COUNT(*)`).

### Капкан №1: Выход за границы массива (`IndexError`)
Если в твоем списке 4 элемента (индексы от 0 до 3), а ты скомандуешь выгрузить `pipes[4]`, конвейер моментально остановится из-за аварии `IndexError: list index out of range`.

### Защита Мастера
Перед точечным обращением по индексу всегда проверяй длину массива через `len()`, либо запрашивай безопасные срезы — срезы никогда не падают в ошибку, даже если ты укажешь границы с избытком, они просто вернут то, что выжило.

## 3. Инструмент №2: Мост неявных связей — Словарь (`Dict`)

### Логика инструмента
Словарь — это коллекция пар «ключ: значение», зажатая в фигурные скобки `{}`. Здесь нет физических индексов (0, 1, 2). Вместо этого ты сам назначаешь уникальный идентификатор (ключ), по которому система мгновенно, без перебора всего массива, находит значение. Это кустарная база данных в RAM.

### Синтаксис
```python
# Справочник лимитов (Маппинг "Материал -> Лимит")
limits_dict = {
    "цемент": 500,
    "арматура": 120,
    "песок": 2000
}
```

### Физика работы
Под капотом словаря лежит механизм **хэш-таблиц**. Доступ к значению по ключу происходит за фиксированное микро-время, независимо от того, лежит в словаре 3 строки или 3 000 000. 
* Ключ обязан быть уникальным и неизменяемым (обычно это текст `str` или число `int`).
* Значением может быть что угодно: число, текст, список или даже другой вложенный словарь.

```python
# Извлечение данных по ключу
current_limit = limits_dict["цемент"]  # Результат: 500

# Добавление новой пары или перезапись старой
limits_dict["щебень"] = 800   # Рождение новой строки
limits_dict["цемент"] = 600   # Перезапись существующего значения
```

### Капкан №1: Поиск несуществующего ключа (`KeyError`)
Если ты попытаешься вытащить лимит по материалу, которого нет в словаре (например, `limits_dict["кабель"]`), Python выбросит критическую ошибку `KeyError`. Весь расчет заморожен.

### Защита Мастера — Перехватчик пустот через метод `.get()`
Забудь про прямое обращение через квадратные скобки, если не уверен в чистоте справочника на 100%. Используй метод `.get(key, default)`. Это **абсолютный аналог «Коалы» (`COALESCE`) для словарей**. Если ключ найден — он вернет значение. Если не найден — он не упадет, а тихо вернет твою заплатку (например, 0).

```python
# Безопасный маневр: если кабеля нет в смете, лимит равен 0
safe_limit = limits_dict.get("кабель", 0)  # Результат: 0 (запрос спасен!)
```

## 4. ПРАКТИЧЕСКИЙ БОЕВОЙ ПОЛИГОН (КОД ДЛЯ ТВОЕГО ВЕРСТАКА)

Скопируй этот код в свой файл `python_master.ipynb` и запусти ячейку. Мы сымитируем работу распределительного узла ПТО: скрипт принимает массив грязных записей о поставках, зачищает их, обращается к справочнику лимитов через безопасный метод `.get()` и формирует итоговый отчет.

```python
# =====================================================================
# СЦЕНАРИЙ: Маппинг грязных поставок через справочник лимитов с защитой от ошибок
# =====================================================================

print("--- ИНИЦИАЛИЗАЦИЯ СБОРЩИКА КОЛЛЕКЦИЙ ---")

# 1. Наш жесткий справочник сметных лимитов (Словарь-мост)
estimate_limits = {
    "арматура а500с": 100.0,
    "цемент м500": 250.0,
    "бетон в25": 500.0
}

# 2. Входящий пакет грязных данных с линии СМР (Список)
# Порядок: [Название_материала, Фактический_объем]
incoming_supply_1 = ["  арматура А500С ", 120.0]
incoming_supply_2 = [" Кабель силовой ", 15.0]  # Этого материала НЕТ в смете!

# ---------------------------------------------------------------------
# ТЕСТИРУЕМ МАНЕВР НА ПОСТАВКЕ №1 (Штатный материал из сметы)
# ---------------------------------------------------------------------
# Ювелирная зачистка ключа
clean_key_1 = incoming_supply_1[0].strip().lower()
fact_vol_1 = incoming_supply_1[1]

# Извлекаем лимит через "Коалу" словаря
limit_1 = estimate_limits.get(clean_key_1, 0.0)

print("Материал: " + clean_key_1.upper())
print("Факт: " + str(fact_vol_1) + " | Лимит: " + str(limit_1))

if fact_vol_1 > limit_1:
    print("РЕЗУЛЬТАТ: Обнаружен перерасход на " + str(fact_vol_1 - limit_1) + " ед.!")
print("-" * 40)

# ---------------------------------------------------------------------
# ТЕСТИРУЕМ МАНЕВР НА ПОСТАВКЕ №2 (Внесистемный материал — Ловушка KeyError заблокирована)
# ---------------------------------------------------------------------
clean_key_2 = incoming_supply_2[0].strip().lower()
fact_vol_2 = incoming_supply_2[1]

# Если бы мы написали estimate_limits[clean_key_2] — скрипт бы умер.
# Метод .get() подставляет заплатку 0.0 для неизвестного материала.
limit_2 = estimate_limits.get(clean_key_2, 0.0)

print("Материал: " + clean_key_2.upper())
print("Факт: " + str(fact_vol_2) + " | Лимит: " + str(limit_2) + " (Вне сметы!)")

if limit_2 == 0.0:
    print("РЕЗУЛЬТАТ: Поставка ЗАБЛОКИРОВАНА. Материал отсутствует в базе сметы!")

print("--- СИНХРОНИЗАЦИЯ КОЛЛЕКЦИЙ ЗАВЕРШЕНА ---")
```
